# 3.31 Webhook 通知集成

## 概述

当场景状态变化（启动、停止、失败、Spot 回收/恢复）时，通过 Webhook 推送富文本消息到团队协作平台。支持 5 个平台：Slack、钉钉、飞书、Discord、企业微信。

## 架构设计

### 模块关系

```
NotificationManager
  ├─ Send()              → 系统桌面通知（macOS/Windows/Linux）
  ├─ webhookMgr.Send()   → Webhook 推送（5 平台并发）
  │
  └─ SendSceneStarted()  ─┐
     SendSceneStopped()   │  每个方法同时触发桌面通知 + Webhook
     SendSceneFailed()    │  Webhook 带颜色参数用于富文本卡片
     SendSpotTerminated() │
     SendSpotRecovered()  │
     SendSpotRecoverFailed()
```

### 数据流

```
用户在 Settings 页面配置 Webhook URL
  → SetWebhookConfig() → gui_settings.json
  → webhookMgr.SetConfig() 热更新内存配置

场景状态变化
  → NotificationManager.SendXxx()
    → nm.Send()               // 桌面通知
    → nm.webhookMgr.Send()    // Webhook（goroutine 内并发推送所有已配置平台）
```

## 支持平台与消息格式

| 平台 | 消息类型 | 签名支持 | 消息样式 |
|------|---------|---------|---------|
| Slack | `attachments` | 无 | 带颜色竖条的附件卡片 |
| 钉钉 | `actionCard` | HMAC-SHA256 | Markdown 卡片 + 底部链接按钮 |
| 飞书 | `interactive` card | HMAC-SHA256 | 带彩色 header 的交互卡片 |
| Discord | `embeds` | 无 | 嵌入式卡片 + 颜色侧边条 |
| 企业微信 | `markdown` | 无 | Markdown 格式消息 |

### 颜色编码

| 事件 | 颜色 | 含义 |
|------|------|------|
| 场景启动 / Spot 恢复成功 | `#36a64f` 绿色 | 成功 |
| 场景停止 | `#ffa500` 橙色 | 警告 |
| 场景失败 / Spot 恢复失败 | `#ff0000` 红色 | 错误 |
| Spot 实例被回收 | `#ff4500` 橙红色 | 严重警告 |

## 关键实现

### WebhookConfig 结构

```go
type WebhookConfig struct {
    Enabled         bool   `json:"enabled"`
    Slack           string `json:"slack"`
    Dingtalk        string `json:"dingtalk"`
    DingtalkSecret  string `json:"dingtalkSecret"`
    Feishu          string `json:"feishu"`
    FeishuSecret    string `json:"feishuSecret"`
    Discord         string `json:"discord"`
    Wecom           string `json:"wecom"`
}
```

此结构定义在 `webhook.go` 中，同时用于后端配置管理和前后端 API 传输。与 `GUISettings` 中的 8 个扁平字段做映射。

### 签名计算

**钉钉（HMAC-SHA256 + URL 参数）**

```go
func (wm *WebhookManager) dingtalkSign(webhookURL, secret string) string {
    timestamp := fmt.Sprintf("%d", time.Now().UnixMilli())
    stringToSign := timestamp + "\n" + secret
    h := hmac.New(sha256.New, []byte(secret))
    h.Write([]byte(stringToSign))
    sign := url.QueryEscape(base64.StdEncoding.EncodeToString(h.Sum(nil)))
    return fmt.Sprintf("%s&timestamp=%s&sign=%s", webhookURL, timestamp, sign)
}
```

签名作为 URL 参数拼接到 Webhook URL 末尾。

**飞书（HMAC-SHA256 + JSON 字段）**

```go
timestamp := fmt.Sprintf("%d", time.Now().Unix())
stringToSign := timestamp + "\n" + secret
h := hmac.New(sha256.New, []byte(stringToSign))
sign := base64.StdEncoding.EncodeToString(h.Sum(nil))
payload["timestamp"] = timestamp
payload["sign"] = sign
```

签名作为 JSON payload 的顶级字段发送。注意飞书用秒级时间戳，钉钉用毫秒级。

### 发送机制

```go
func (wm *WebhookManager) Send(title, message, color string) {
    cfg := wm.GetConfig()
    if !cfg.Enabled {
        return
    }
    go func() {
        // 依次检查 5 个平台的 URL，非空则发送
        // 每个平台独立错误处理，单个失败不影响其他平台
    }()
}
```

- 发送在独立 goroutine 中进行，不阻塞调用方
- HTTP 客户端超时 10 秒
- 错误只记日志，不影响主流程

### 测试发送

```go
func (wm *WebhookManager) TestWebhook(platform, webhookURL, secret string) error {
```

前端每个平台输入框旁有测试按钮，点击后调用此函数，实时验证 URL 和签名是否正确。

### 前端 UI 结构

```
Webhook 通知
  [主开关]
  ├─ Slack Webhook URL      [输入框] [测试]
  ├─ 钉钉 Webhook URL       [输入框] [测试]
  │   签名密钥（可选）        [输入框]
  ├─ 飞书 Webhook URL       [输入框] [测试]
  │   签名密钥（可选）        [输入框]
  ├─ Discord Webhook URL    [输入框] [测试]
  ├─ 企业微信 Webhook URL    [输入框] [测试]
  └─ [保存] [状态提示]
```

- 主开关关闭时折叠所有配置项
- 前端通过 `$effect` 自动加载已有配置
- 保存调用 `SetWebhookConfig()`，同步更新 `gui_settings.json` 和内存中的 `WebhookManager`

## 配置存储

所有 Webhook 配置存储在 `gui_settings.json` 中：

```json
{
  "webhookEnabled": true,
  "webhookSlack": "https://hooks.slack.com/services/xxx",
  "webhookDingtalk": "https://oapi.dingtalk.com/robot/send?access_token=xxx",
  "webhookDingtalkSecret": "SECxxxxxxx",
  "webhookFeishu": "",
  "webhookFeishuSecret": "",
  "webhookDiscord": "",
  "webhookWecom": ""
}
```

URL 为空的平台不会触发发送。

## 注意事项

1. **热更新**：`SetWebhookConfig()` 同时写文件和更新 `WebhookManager` 内存配置，无需重启应用
2. **签名密钥安全**：密钥存储在本地 `gui_settings.json`，不会上传或同步
3. **并发安全**：`WebhookManager` 使用 `sync.RWMutex` 保护配置读写
4. **错误隔离**：单个平台发送失败不影响其他平台，错误通过 `gologger.Error()` 记录
5. **与桌面通知的关系**：两套通知系统独立运行，Webhook 开关不影响系统桌面通知，反之亦然
