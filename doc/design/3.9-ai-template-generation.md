# AI 模板生成功能设计文档

## 概述

基于 LLM（大语言模型）的模板生成功能允许用户通过自然语言描述需求，AI 自动生成完整的 RedC 场景模板（包括 Terraform 文件和 case.json），并保存到指定的模板目录。

**版本**：v3.0.8+  
**作者**：RedC Team  
**最后更新**：2026-03-08

---

## 1. 功能定位

### 1.1 用户需求
- 降低 RedC 场景模板的创建门槛
- 无需手动编写 Terraform 文件
- 快速生成符合 RedC 规范的场景模板
- 支持多云厂商（AWS、阿里云、腾讯云等）

### 1.2 功能位置
- **GUI 页面**：AI 集成（AI Integration）
- **入口**：侧边栏 → AI 集成 → AI 模板生成
- **位置**：在"智能场景推荐"功能上方

## 2. 技术架构

### 2.1 整体流程

```
用户输入需求
    ↓
前端发送请求到后端
    ↓
后端调用 AI API（OpenAI/Anthropic）
    ↓

前端解析 Markdown 提取文件内容
    ↓
用户点击"保存到模板"
    ↓
后端保存到 ai-templates 目录
    ↓
显示成功提示
```

## 5. 文件解析规则

### 5.1 Markdown 解析

**输入格式**：
```markdown
### case.json
{
  "name": "aws-ubuntu-target-8080",
  "description": "AWS Ubuntu 22.04 with port 8080",
  ...
}

### main.tf
resource "aws_instance" "target" {
  ...
}

### variables.tf
variable "instance_type" {
  ...
}
```

**解析逻辑**：
```javascript
function parseTemplateMarkdown(markdown) {
    const files = {};
    const fileBlocks = markdown.split(/^###\s+/m);
    
    for (const block of fileBlocks) {
        const lines = block.split('\n');
        const filename = lines[0].trim();
        
        // 验证文件扩展名
        if (!filename.match(/\.(json|tfvars|tf|md|sh|yaml|yml)$/i)) {
            continue;
        }
        
        // 提取内容并移除代码块标记
        const content = lines.slice(1).join('\n').trim();
        let fileContent = content.replace(/^```[\w]*\n?/g, '')
                                  .replace(/```$/g, '')
                                  .trim();
        
        files[filename] = fileContent;
    }
    
    return files;
}
```

### 5.2 支持的文件类型

- `*.json` - JSON 配置文件
- `*.tf` - Terraform 文件
- `*.tfvars` - Terraform 变量文件
- `*.md` - Markdown 文档
- `*.sh` - Shell 脚本
- `*.yaml` / `*.yml` - YAML 配置
- `userdata` - Userdata 脚本

## 8. 流式响应处理

### 8.1 后端事件发送

**app_ai.go**：
```go
err = client.ChatStream(ctx, messages, func(chunk string) error {
    runtime.EventsEmit(a.ctx, "ai-template-gen-chunk", chunk)
    return nil
})

// 发送完成信号
runtime.EventsEmit(a.ctx, "ai-template-gen-complete", true)
```

### 8.2 前端事件监听

**AIIntegration.svelte**：
```javascript
onMount(() => {
    EventsOn('ai-template-gen-chunk', (chunk) => {
        generatedTemplate += chunk;
    });
    
    EventsOn('ai-template-gen-complete', () => {
        templateGenerating = false;
    });
});

// 清理监听
setTimeout(() => {
    EventsOff('ai-template-gen-chunk');
    EventsOff('ai-template-gen-complete');
}, 1000);
```

## 9. 错误处理

### 9.1 常见错误

1. **AI 未配置**
   - 检测：`isAIConfigured()` 返回 false
   - 处理：显示"AI 服务未配置"提示，引导用户配置

2. **解析失败**
   - 检测：`Object.keys(files).length === 0`
   - 处理：显示"无法解析模板，请确保生成的内容包含文件标题"

3. **保存失败**
   - 检测：`SaveTemplateFiles` 抛出异常
   - 处理：显示错误信息，保留生成的内容

### 9.2 错误提示

```javascript
try {
    const savedPath = await SaveTemplateFiles(templateName, files);
    successMessage = `模板已保存到：${savedPath}`;
} catch (e) {
    error = e.message || String(e);
    successMessage = '';
}
```

## 10. 最佳实践

### 10.1 Prompt 编写建议

1. **明确云厂商**：指定 AWS、阿里云等
2. **描述系统**：Ubuntu 22.04、CentOS 7 等
3. **说明用途**：Web 服务器、数据库、测试环境等
4. **指定端口**：80、443、8080 等
5. **特殊需求**：公网 IP、特定配置等

**示例**：
```
我需要一个阿里云的 Ubuntu 22.04 测试机，开放 8080 端口，用于 API 测试
```

### 10.2 模板验证

保存后建议用户：
1. 检查 case.json 的 name 字段是否正确
2. 验证 main.tf 的资源定义是否完整
3. 确认 variables.tf 包含必要的变量
4. 测试 terraform init 和 terraform plan

### 10.3 模板优化

如果生成的模板不理想：
1. 重新生成，提供更详细的描述
2. 手动编辑生成的文件
3. 保存到本地模板后进行二次修改
