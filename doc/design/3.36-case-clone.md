# 3.36 场景克隆功能

## 背景

用户在使用 redc-gui 时，经常需要创建配置相似的多个场景。手动重新选择模板、填写相同参数效率低下。场景克隆功能允许一键复制已有场景的配置，快速创建相似场景。

## 设计方案

### 两种克隆类型

| 类型 | 源数据 | 克隆方式 |
|------|--------|----------|
| 预定义场景 | Case（模板 + 参数） | 从相同模板重新创建，复用原有参数 |
| 自定义部署 | CustomDeployment（DeploymentConfig） | 深拷贝 DeploymentConfig 创建新部署 |

### 后端实现

#### `CloneCase(caseID string)` — app_scene.go

1. 通过 `LoadProjectCases` 查找源场景
2. 从源场景 `Parameter` 字段解析出 `key=value` 变量
3. 使用源场景的 `Type`（模板名）+ 变量调用 `CaseCreate`
4. 新名称 = 原名称 + "-clone"
5. 异步执行（go routine），完成后 emitRefresh

```go
func (a *App) CloneCase(caseID string) error {
    // 查找源场景
    source := findCase(caseID)
    // 解析参数
    vars := parseParams(source.Parameter)
    // 异步创建
    go project.CaseCreate(source.Type, user, source.Name+"-clone", vars)
}
```

#### `CloneCustomDeployment(deploymentID string)` — app_deployment_extra.go

1. 通过 `ListCustomDeployments` 查找源部署
2. 深拷贝 `DeploymentConfig`（包括 Provider、Region、InstanceType、Userdata、Variables 等）
3. 新名称 = 原配置名 + "-clone"
4. 调用 `CreateCustomDeployment` 创建

```go
func (a *App) CloneCustomDeployment(deploymentID string) (*CustomDeployment, error) {
    source := findDeployment(deploymentID)
    cloneConfig := deepCopy(source.Config)
    cloneConfig.Name += "-clone"
    return service.CreateCustomDeployment(cloneConfig, ...)
}
```

### 前端实现

在预定义场景（Cases.svelte）和自定义部署（CustomDeploymentList.svelte）的操作列中，在标签按钮旁新增克隆图标按钮（紫色 hover 效果），使用文档复制图标。

按钮仅在场景不处于中间态（starting/stopping/removing）时显示。
