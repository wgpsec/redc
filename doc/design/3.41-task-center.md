# 3.41 任务中心（Task Center）

## 概述

任务中心是从预定义场景（Cases）页面中拆分出来的独立运维页面，提供定时任务和周期性任务的统一管理能力。它位于侧边栏「运维工具」分组下，与控制台、资源管理、SSH 终端管理并列。

支持 4 种任务操作：
- **启动场景**（start）— 定时启动指定场景
- **停止场景**（stop）— 定时停止指定场景
- **SSH 命令执行**（ssh_command）— 定时在场景实例上执行 SSH 命令
- **自动停机**（auto_stop）— N 小时后自动停止场景（成本控制）

## 设计动机

- 预定义场景页面职责过重，同时承担场景管理和定时任务管理
- 定时任务是运维能力，应归入运维工具分组
- 用户需要周期性任务（每天/每周/自定义间隔）支持，原有的一次性定时任务无法满足
- 用户需要定时执行运维命令（日志清理、安全更新、健康检查）
- 用户需要成本控制能力，防止忘关实例
- 任务执行结果需要通知到 IM 平台

## 后端设计

### ScheduledTask 结构

```go
type ScheduledTask struct {
    ID             string    `json:"id"`
    CaseID         string    `json:"caseId"`
    CaseName       string    `json:"caseName"`
    Action         string    `json:"action"`       // "start" | "stop" | "ssh_command" | "auto_stop"
    ScheduledAt    time.Time `json:"scheduledAt"`
    Status         string    `json:"status"`        // "pending" | "completed" | "failed" | "cancelled"
    Error          string    `json:"error"`
    RepeatType     string    `json:"repeatType"`    // "once" | "daily" | "weekly" | "interval"
    RepeatInterval int       `json:"repeatInterval"` // 分钟数（仅 interval 模式使用）
    CompletedAt    *time.Time `json:"completedAt"`
    SSHCommand     string    `json:"sshCommand"`     // SSH 命令内容（仅 ssh_command 模式）
    TaskResult     string    `json:"taskResult"`     // 执行结果（如 SSH 输出）
    NotifyEnabled  bool      `json:"notifyEnabled"`  // 完成时发送通知
}
```

### 数据库迁移

使用 `ALTER TABLE ADD COLUMN` 方式添加新字段，SQLite 在列已存在时静默失败，保证向后兼容：

```go
func (ts *TaskScheduler) InitDB() {
    // ... 原有建表逻辑 ...
    
    // 迁移：添加新字段
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN repeat_type TEXT DEFAULT 'once'")
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN repeat_interval INTEGER DEFAULT 0")
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN completed_at DATETIME")
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN ssh_command TEXT DEFAULT ''")
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN task_result TEXT DEFAULT ''")
    ts.db.Exec("ALTER TABLE scheduled_tasks ADD COLUMN notify_enabled INTEGER DEFAULT 0")
}
```

### 周期任务自动续期

任务执行完成后（无论成功或失败），`executeTask()` 调用 `scheduleNextRepeat()`：

```go
func (ts *TaskScheduler) scheduleNextRepeat(task *ScheduledTask) {
    if task.RepeatType == "once" || task.RepeatType == "" {
        return
    }
    
    var nextTime time.Time
    switch task.RepeatType {
    case "daily":
        nextTime = task.ScheduledAt.Add(24 * time.Hour)
    case "weekly":
        nextTime = task.ScheduledAt.Add(7 * 24 * time.Hour)
    case "interval":
        nextTime = task.ScheduledAt.Add(time.Duration(task.RepeatInterval) * time.Minute)
    }
    
    // 快进到未来时间（如果计算出的下一次时间已过去）
    for nextTime.Before(time.Now()) {
        switch task.RepeatType {
        case "daily":
            nextTime = nextTime.Add(24 * time.Hour)
        case "weekly":
            nextTime = nextTime.Add(7 * 24 * time.Hour)
        case "interval":
            nextTime = nextTime.Add(time.Duration(task.RepeatInterval) * time.Minute)
        }
    }
    
    // 创建新任务（新 ID，继承 repeat 配置）
    ts.AddTaskWithRepeat(task.CaseID, task.CaseName, task.Action, nextTime,
        task.RepeatType, task.RepeatInterval)
}
```

### 核心 API

| 函数 | 签名 | 描述 |
|------|------|------|
| `AddTaskFull` | `(caseID, caseName, action string, scheduledAt time.Time, repeatType string, repeatInterval int, sshCommand string, notifyEnabled bool)` | 创建完整配置的任务 |
| `AddTaskWithRepeat` | 代理到 `AddTaskFull`（sshCommand="", notifyEnabled=false） | 向后兼容 |
| `ListAllTasksFromDB` | `() []ScheduledTask` | 从 DB 读取所有任务（含最近 7 天历史），按 scheduled_at 降序 |
| `ScheduleTaskFull` | App 层封装 | Wails 绑定，调用 `AddTaskFull` |
| `ListAllScheduledTasks` | App 层封装 | Wails 绑定，调用 `ListAllTasksFromDB` |

### 任务执行分发逻辑

`executeTask()` 根据 `action` 分发到不同的回调：

```go
switch task.Action {
case "ssh_command":
    result, err = s.onSSHCommand(task.CaseID, task.SSHCommand)
case "auto_stop":
    err = s.onExecute(task.CaseID, "stop")
default: // "start" / "stop"
    err = s.onExecute(task.CaseID, task.Action)
}
```

执行完成后：
1. 更新状态（completed/failed）和 TaskResult
2. 如果 `NotifyEnabled` 为 true，调用 `onNotify` 回调发送通知
3. 调用 `scheduleNextRepeat` 为周期任务创建下一次执行

### 回调注册（app.go）

```go
// SSH 命令回调
a.taskScheduler.SetSSHCommandCallback(func(caseID, command string) (string, error) {
    c, _ := a.project.GetCase(caseID)
    sshConfig, _ := c.GetSSHConfig()
    result := a.execSSHCommand(sshConfig, command)
    // 返回 stdout + stderr，失败时返回 error
})

// 通知回调
a.taskScheduler.SetNotifyCallback(func(title, message string) {
    a.notificationMgr.Send(title, message)      // 系统通知
    a.notificationMgr.webhookMgr.Send(...)       // Webhook
})
```

### TaskResult 存储

- SSH 命令执行结果保存到 `task_result` 字段
- 数据库存储时截断至 4KB（`updateTaskResultInDB`）
- 前端历史记录中显示「查看结果」按钮，点击弹出 modal 查看完整输出

### ListAllTasksFromDB 与内存 map 的区别

- `ts.tasks` (内存 map)：仅保存 `pending` 状态的任务，用于调度循环
- `ListAllTasksFromDB()`：直接读 SQLite，包含 completed/failed/cancelled 历史记录（最近 7 天）

## 与 Cases 页面的关系

- Cases 页面保留 `ScheduleDialog` 组件（单个场景的快捷定时按钮）
- Cases 页面移除了 `ScheduledTasksManager` 嵌入组件

- 所有任务统一在任务中心页面查看和管理

## 注意事项

1. **SQLite 并发**：`TaskScheduler` 使用 `sync.Mutex` 保护所有读写操作
2. **时间快进**：周期任务如果程序关闭期间错过了多次执行，续期时会快进到下一个未来时间点，不会补执行
3. **ListCases 返回格式**：场景的 ID/Name 字段存在大小写不一致（`id`/`Id`, `name`/`Name`），TaskCenter 前端通过 `c.id || c.Id` 兼容处理
4. **Svelte 5 限制**：不能在任意位置使用 `{@const}`，需要在 `{#each}` / `{#if}` 块内部或改为内联表达式
5. **DB 迁移安全性**：`ALTER TABLE ADD COLUMN` 在 SQLite 中如果列已存在会返回错误，但 `db.Exec` 忽略该错误，保证重复启动安全
6. **SSH 命令执行安全**：SSH 命令通过已有的 `app.execSSHCommand` 执行，复用场景的 SSH 配置（IP/端口/凭据），场景必须处于运行状态
7. **TaskResult 截断**：SSH 命令输出在写入数据库前截断至 4KB，防止大量输出占满数据库
8. **自动停机是一次性的**：`auto_stop` action 强制 `repeatType = "once"`，前端隐藏周期配置。需要重复自动停机需手动创建 `stop` + 周期任务
9. **通知回调**：`onNotify` 同时发送系统通知（macOS/Windows/Linux）和 Webhook（钉钉/飞书/Slack/Discord/企业微信），受全局通知开关控制
10. **notify_enabled 存储**：SQLite 没有 boolean 类型，使用 INTEGER（0/1）存储，Go 层做 int↔bool 转换
