# 更新日志功能

## 概述

在关于页面添加更新日志显示，方便用户了解版本变更历史。更新日志使用 JSON 文件维护，便于开发者快速添加新功能记录。

---

## 1. 实现方式

### 1.1 更新日志文件

**文件**: `frontend/public/changelog.json`

```json
{
  "changelog": [
    {
      "version": "3.0.0",
      "date": "2025-02-25",
      "changes": [
        "新增：自定义部署功能",
        "新增：AI 分析部署错误功能",
        "新增：本地模板导入导出功能",
        "新增：右键菜单禁用配置持久化",
        "新增：GUI 配置文件 (gui_settings.json)",
        "优化：模板类型从布尔值改为枚举",
        "优化：预定义场景改名为预定义场景"
      ]
    }
  ]
}
```

### 1.2 数据结构说明

| 字段 | 类型 | 说明 |
|------|------|------|
| version | string | 版本号，如 "3.0.0" |
| date | string | 发布日期，格式 "YYYY-MM-DD" |
| changes | string[] | 变更内容列表 |

## 2. 前端实现

### 2.1 About.svelte 修改

**文件**: `frontend/src/components/About/About.svelte`

```svelte
<script>
  import { onMount } from 'svelte';
  
  let { t } = $props();
  
  let changelog = $state([]);
  let loading = $state(true);
  
  onMount(async () => {
    try {
      const res = await fetch('/changelog.json');
      const data = await res.json();
      changelog = data.changelog || [];
    } catch (e) {
      console.error('Failed to load changelog:', e);
    } finally {
      loading = false;
    }
  });
</script>

<!-- 更新日志 UI -->
<div class="bg-white rounded-xl border border-gray-100 p-6">
  <h2 class="text-lg font-semibold text-gray-900 mb-4">
    {t.changelog || '更新日志'}
  </h2>
  
  {#if loading}
    <div class="text-sm text-gray-500">{t.loading || '加载中...'}</div>
  {:else if changelog.length === 0}
    <div class="text-sm text-gray-500">{t.noChangelog || '暂无更新日志'}</div>
  {:else}
    <div class="space-y-4">
      {#each changelog as item}
        <div class="border-l-2 border-gray-200 pl-4">
          <div class="flex items-center gap-2 mb-2">
            <span class="px-2 py-0.5 bg-blue-100 text-blue-700 text-xs font-medium rounded">
              {item.version}
            </span>
            <span class="text-xs text-gray-500">{item.date}</span>
          </div>
          <ul class="space-y-1">
            {#each item.changes as change}
              <li class="text-sm text-gray-600 flex items-start gap-2">
                <span class="text-gray-400 mt-1">•</span>
                <span>{change}</span>
              </li>
            {/each}
          </ul>
        </div>
      {/each}
    </div>
  {/if}
</div>
```

## 3. 维护指南

### 3.1 添加新版本

每次发布新版本时，在 `changelog.json` 的 `changelog` 数组**最前面**添加新条目：

```json
{
  "changelog": [
    {
      "version": "X.X.X",
      "date": "YYYY-MM-DD",
      "changes": [
        "新增：xxx功能",
        "修复：xxx问题",
        "优化：xxx体验"
      ]
    },
    {
      "version": "3.0.0",
      "date": "2025-02-25",
      "changes": [...]
    }
  ]
}
```

### 3.2 变更类型前缀

建议使用以下前缀区分变更类型：
- `新增：` - 新功能
- `修复：` - Bug 修复
- `优化：` - 性能或体验优化
- `移除：` - 移除的功能
- `重构：` - 代码重构

## 5. 注意事项

1. **文件位置**: 必须是 `public/changelog.json`，确保构建时能直接访问
2. **日期格式**: 使用 "YYYY-MM-DD" 格式
3. **版本顺序**: 最新版本放最前面
4. **加载失败**: 组件有容错处理，加载失败会显示"暂无更新日志"
