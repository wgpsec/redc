# 3.32 场景标签/分组功能

## 概述

支持给预定义场景（Case）和自定义部署（CustomDeployment）打标签（如 "项目A"、"内网渗透"），并按标签筛选列表。标签存储在 `gui_settings.json` 中，通过 ID 映射，不侵入原有数据模型。

## 设计决策

### 为什么不修改 Protobuf

Case 数据通过 Protobuf 序列化存储在 BoltDB 中（`proto/redc.proto` → `pb/redc.pb.go`）。修改 proto schema 需要：
1. 添加字段
2. 重新生成 Go 代码（`protoc`）
3. 处理数据迁移

而标签是 GUI 层面的轻量元数据，与核心业务逻辑无关。因此选择存储在 `gui_settings.json` 中，以 `map[string][]string` 形式按 ID 映射。

### 统一 ID 命名空间

Case ID 和 CustomDeployment ID 不会冲突（不同生成逻辑），因此共用同一个 `caseTags` map，简化实现。

## 数据结构

### gui_settings.json

```json
{
  "caseTags": {
    "74c78e94292a": ["项目A", "内网渗透"],
    "abc123def456": ["项目B"]
  }
}
```

### GUISettings 结构体

```go
type GUISettings struct {
    // ...existing fields...
    CaseTags map[string][]string `json:"caseTags,omitempty"`
}
```

## 后端 API

| 函数 | 签名 | 说明 |
|------|------|------|
| `SetCaseTags` | `(id string, tags []string) error` | 设置指定 ID 的标签列表（空列表则删除 key） |
| `GetAllCaseTags` | `() map[string][]string` | 返回完整标签映射 |
| `GetAllTagNames` | `() []string` | 返回所有唯一标签名（用于筛选栏和自动补全） |

`ListCases()` 返回的 `CaseInfo` 新增 `Tags []string` 字段，在查询时从 `gui_settings.json` 填充。

`ListCustomDeployments()` 未修改返回类型（`*redc.CustomDeployment` 在 mod 包中），前端通过单独调用 `GetAllCaseTags()` 获取标签。

## 前端实现

### 标签筛选栏

位于列表表格顶部，仅在存在标签时显示：

```
标签: [全部] [项目A] [内网渗透] [项目B]
```

- 点击标签高亮筛选，再次点击取消
- "全部" 按钮重置筛选
- 使用 `$derived` 计算 `filteredCases` / `filteredDeployments`

### 标签显示

标签以彩色小药丸（pill）形式显示在场景名称右侧：

```
modest_chinchilla_aliyun_pro  [项目A] [内网渗透]
```

颜色通过标签名 hash 自动分配（10 种预设颜色循环）。

### 标签编辑

操作列新增标签图标按钮（🏷），点击展开内联编辑行：

```
标签: [项目A ×] [内网渗透 ×]  [输入标签名] [+]
```

- 输入框支持 Enter 快捷添加
- `<datalist>` 提供已有标签的自动补全建议
- 点击 × 删除单个标签
- 修改即时保存到 `gui_settings.json`

## 注意事项

1. **即时保存**：标签增删立即写入 `gui_settings.json`，无需手动保存按钮
2. **自动补全**：使用 HTML `<datalist>` 原生补全，跨页面共享已有标签
3. **筛选联动**：切换标签时清空批量选择状态，避免选中不可见条目
4. **颜色一致性**：同名标签在两个页面中显示相同颜色（基于 hash 确定性算法）
5. **数据清理**：删除 Case 时不自动清理 `caseTags` 中的对应条目（惰性清理，不影响功能）
