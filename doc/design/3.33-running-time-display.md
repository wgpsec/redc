# 3.33 场景运行时间统计

## 概述

当场景处于 `running` 状态时，在时间列旁显示已运行时长（如 `⏱ 2h 15m`），每分钟自动刷新。纯前端实现，无后端改动。

## 设计决策

### 为什么是纯前端

运行时间 = `当前时间 - 状态变更时间`。后端已在状态切换时记录 `StateTime`（Case）和 `UpdatedAt`（CustomDeployment），前端用 `Date.now() - Date.parse(stateTime)` 即可计算，无需增加 API。

### 为什么用 1 分钟刷新间隔

对于基础设施管理工具，秒级精度没有实际意义。1 分钟刷新（`setInterval(60000)`）在实用性和性能之间取得平衡。页面切换或手动刷新也会触发更新。

## 实现

### 核心逻辑

```javascript
let nowTick = $state(Date.now());
let tickTimer = null;

function formatElapsed(fromStr) {
  const from = new Date(fromStr).getTime();
  const diff = Math.max(0, Math.floor((nowTick - from) / 1000));
  const d = Math.floor(diff / 86400);
  const h = Math.floor((diff % 86400) / 3600);
  const m = Math.floor((diff % 3600) / 60);
  if (d > 0) return `${d}d ${h}h ${m}m`;
  if (h > 0) return `${h}h ${m}m`;
  return `${m}m`;
}

// onMount
tickTimer = setInterval(() => { nowTick = Date.now(); }, 60000);

// onDestroy
clearInterval(tickTimer);
```

`nowTick` 是 Svelte 5 的 `$state` 响应式变量。每分钟更新一次，触发所有引用它的 `formatElapsed()` 重新计算。

### 时间来源

| 页面 | 时间字段 | 格式 | 说明 |
|------|---------|------|------|
| 预定义场景 (Cases) | `c.stateTime` | RFC3339 | `time.Now().Format(time.RFC3339)` |
| 自定义部署 (CustomDeployment) | `deployment.updated_at` | ISO 8601 | `time.Time` JSON 序列化 |

两种格式都能被 `new Date()` 正确解析。

### 显示格式

| 运行时长 | 显示 |
|---------|------|
| < 1 小时 | `0m` / `15m` / `59m` |
| 1 小时 ~ 1 天 | `1h 30m` |
| ≥ 1 天 | `2d 5h 30m` |

### UI 位置

- **预定义场景**：时间列（stateTime）右侧，绿色文字 `⏱ Xh Xm`
- **自定义部署**：创建时间列右侧，同样绿色文字

仅在 `state === 'running'` 时显示。

## 注意事项

1. **性能**：定时器仅更新一个数字变量，不触发 DOM 全量重绘，Svelte 的细粒度响应式确保只更新相关文本节点
2. **时区**：`stateTime` 是 RFC3339（含时区），`new Date()` 自动处理时区转换
3. **清理**：组件销毁时清除定时器，避免内存泄漏
4. **边界值**：`Math.max(0, ...)` 防止时钟偏差导致负值
