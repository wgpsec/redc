# MCP 工具扩展设计文档

## 1. 背景

redc MCP Server v3.0.7 提供了 18 个基础工具，覆盖模板管理、Case 生命周期、SSH/文件传输和配置查询。但 App 层已有的 Compose 编排、成本估算、自定义部署、项目管理、定时任务等能力未暴露为 MCP 工具。

v3.0.8 新增 19 个工具，总计 37 个，实现 MCP 工具对 redc 全功能的覆盖。

## 2. 架构设计

### 2.1 核心问题：循环依赖

```
main 包 (App)  ←→  mod/mcp 包 (MCPServer)
     ↑                    ↑
     │  不能互相 import    │
     └────────────────────┘
```

`MCPServer` 在 `mod/mcp` 包中，`App` 在 `main` 包中。Go 禁止循环导入，因此 MCP 不能直接引用 App 的方法。

### 2.2 解决方案：AppBridge 接口

```
┌──────────────────────────────────────────┐
│  main 包                                  │
│  ┌──────────────────────────────────────┐│
│  │ App struct                            ││
│  │  实现 AppBridge 接口 (app_mcp_bridge.go) ││
│  └──────────────────────────────────────┘│
│          │ 传入                            │
│          ▼                                │
│  NewMCPServer(project, app)              │
│  NewMCPServerManager(project, app)       │
└──────────────────────────────────────────┘
              │
              ▼
┌──────────────────────────────────────────┐
│  mod/mcp 包                               │
│  ┌──────────────────────────────────────┐│
│  │ AppBridge interface  (bridge.go)      ││
│  │  19 个方法签名                         ││
│  └──────────────────────────────────────┘│
│  ┌──────────────────────────────────────┐│
│  │ MCPServer struct                      ││
│  │  app AppBridge  // 可能为 nil (CLI)    ││
│  └──────────────────────────────────────┘│
│  tools_compose.go   (3 tools)            │
│  tools_cost.go      (6 tools)            │
│  tools_deployment.go(3 tools)            │
│  tools_project.go   (5 tools)            │
│  tools_scheduler.go (3 tools)            │
└──────────────────────────────────────────┘
```

**关键设计点：**
- `AppBridge` 方法返回 `interface{}` 而非具体类型，避免引用 `main` 包类型
- CLI 模式传 `nil`，扩展工具在 `getTools()` 中检查 `s.app != nil` 才注册
- 每个工具方法开头检查 `s.app == nil`，返回 "requires GUI mode" 错误

## 3. 工具清单

### 3.1 Compose 编排（3 个）

| 工具 | 参数 | 说明 |
|------|------|------|
| `compose_preview` | `file?`, `profiles?` | 预览服务列表和依赖关系 |
| `compose_up` | `file?`, `profiles?` | 启动部署（异步，立即返回） |
| `compose_down` | `file?`, `profiles?` | 销毁部署（异步，立即返回） |

注意：`compose_up` / `compose_down` 是异步操作（Go 协程），MCP 工具立即返回 "started" 状态。

### 3.2 成本与资源（6 个）

| 工具 | 参数 | 说明 |
|------|------|------|
| `get_cost_estimate` | `template` (必需) | 估算模板部署成本 |
| `get_balances` | `providers?` | 查询云账户余额 |
| `get_resource_summary` | 无 | 各云厂商资源汇总 |
| `get_predicted_monthly_cost` | 无 | 当月预估成本 |
| `get_bills` | `providers?` | 云账单信息 |
| `get_total_runtime` | 无 | 活跃场景总运行时长 |

### 3.3 自定义部署（3 个）

| 工具 | 参数 | 说明 |
|------|------|------|
| `list_deployments` | 无 | 列出所有自定义部署 |
| `start_deployment` | `deployment_id` (必需) | 启动部署 |
| `stop_deployment` | `deployment_id` (必需) | 停止部署 |

### 3.4 项目/Profile 管理（5 个）

| 工具 | 参数 | 说明 |
|------|------|------|
| `list_projects` | 无 | 列出所有项目 |
| `switch_project` | `project_name` (必需) | 切换项目 |
| `list_profiles` | 无 | 列出配置档案 |
| `get_active_profile` | 无 | 获取当前配置 |

### 3.5 定时任务（3 个）

## 4. 工具注册机制

### 4.1 Schema 注册

每个 `tools_*.go` 文件导出 `xxxToolSchemas() []Tool` 函数。在 `mcp.go` 的 `getTools()` 中：

```go
func (s *MCPServer) getTools() []Tool {
    tools := []Tool{...existing 18 tools...}

    if s.app != nil {
        tools = append(tools, composeToolSchemas()...)
        tools = append(tools, costToolSchemas()...)
        tools = append(tools, deploymentToolSchemas()...)
        tools = append(tools, projectToolSchemas()...)
        tools = append(tools, schedulerToolSchemas()...)
    }

    return tools
}
```

### 4.2 路由注册

在 `executeTool()` 的 switch 语句中新增 19 个 case 分支，按类别分组，每个 case 解析参数后调用对应的 `s.toolXxx()` 方法。

## 5. AppBridge 接口定义

```go
type AppBridge interface {
    // Compose
    MCPComposePreview(file string, profiles []string) (interface{}, error)
    MCPComposeUp(file string, profiles []string) error
    MCPComposeDown(file string, profiles []string) error

    // Cost
    MCPGetCostEstimate(template string) (interface{}, error)
    MCPGetBalances(providers []string) (interface{}, error)
    MCPGetResourceSummary() (interface{}, error)
    MCPGetPredictedMonthlyCost() (interface{}, error)
    MCPGetBills(providers []string) (interface{}, error)
    MCPGetTotalRuntime() (interface{}, error)

    // Deployment
    MCPListCustomDeployments() (interface{}, error)
    MCPStartCustomDeployment(id string) error
    MCPStopCustomDeployment(id string) error

    // Project/Profile
    MCPListProjects() (interface{}, error)
    MCPSwitchProject(name string) error
    MCPListProfiles() (interface{}, error)
    MCPGetActiveProfile() (interface{}, error)
    MCPSetActiveProfile(id string) error

    // Scheduler
    MCPScheduleTask(caseID, caseName, action string, at time.Time) (interface{}, error)
    MCPListScheduledTasks() interface{}
    MCPCancelScheduledTask(id string) error
}
```

## 6. CLI vs GUI 行为差异

| 场景 | AppBridge | 可用工具数 |
|------|-----------|----------|
| CLI (`redc mcp stdio/sse`) | `nil` | 18（基础工具） |
| GUI (redc-gui 内建 MCP) | `*App` | 37（全部工具） |

CLI 模式下不注册扩展工具（`getTools()` 中 `s.app != nil` 判断），也不出现在 `tools/list` 响应中。即使通过其他方式调用，`executeTool()` 的 default 分支会返回 "unknown tool" 错误。

## 7. 调用者变更

| 调用位置 | 文件 | 变更 |
|----------|------|------|
| GUI 内建 MCP | `app_mcp_scheduler.go` | `NewMCPServerManager(project, a)` — `a` 为 `*App` |
| AI Chat Agent | `app_ai_chat.go` | `NewMCPServer(project, a)` — `a` 为 `*App` |
| CLI MCP 命令 | `cmd/mcp.go` | `NewMCPServerManager(project, nil)` — 无 App |

## 8. 未来扩展

新增 MCP 工具只需：
1. 在 `AppBridge` 接口添加方法签名
2. 在 `app_mcp_bridge.go` 实现委托
3. 创建 `tools_xxx.go`，实现 schema 和执行逻辑
4. 在 `getTools()` / `executeTool()` 注册

无需修改 MCP 协议层代码。
